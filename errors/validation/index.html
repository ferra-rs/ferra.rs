validation

Validation Error

400 Bad Request

Returned when an inbound JSON body or path parameter fails the framework's typed deserialisation, when an Id parameter fails to parse, or when a validator-derived field check rejects the input. The optional errors map contains a list of messages per field name (Map<String, Vec<String>>).

Example RFC 7807 body

{
  "type":   "https://ferra.rs/errors/validation",
  "title":  "Validation Error",
  "status": 400,
  "detail": "validation failed",
  "errors": {
    "title": ["must not be empty"],
    "year":  ["must be between 1888 and the current year"]
  }
}
